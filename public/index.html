What it can do

Built for real site ownership.

Virtual hosts

Use nginx-style per-domain files with server names, roots, routes, PHP, proxy fallback, redirects, headers, and SNI cert paths.

Reverse proxy

Proxy to one upstream or a pool with retries, passive ejection, circuit breaker recovery, slow start, keep-alive reuse, and load-balancing policy.

Static performance

Serve static files with sendfile on Darwin, bounded buffered fallback, precompressed sidecars, ETags, cache headers, and byte ranges.

PHP hosting

Run PHP through php-cgi or speak FastCGI directly to php-fpm with pooled worker connections and framework front-controller support.

Observability

Expose Prometheus-style metrics, structured access logs, admin routes/certs/status views, and config validation from the running process.

Protocol work

HTTP/1.1 and native HTTP/2 serve the router today; HTTP/3 QUIC/TLS is in-tree and expanding beyond the default response path.

This page is honest about what is finished: Layerline can already serve normal HTTP/1.1 and HTTP/2 routes, own TLS, proxy apps, host PHP, and expose admin diagnostics. Full HTTP/3 route dispatch and hot reload are still roadmap work.

Setup

Build it, point a site at it, enable the admin UI when you need it.

git clone https://github.com/raya-ac/Layerline.git
cd Layerline
zig build -Doptimize=ReleaseFast
./zig-out/bin/layerline --config server.conf
# server.conf
host = 0.0.0.0
port = 443
dir = public
serve_static_root = true
domain_config_dir = domains-enabled
tls = true
http_redirect = true
# domains-enabled/example.conf
server_name = example.com www.example.com
root = /srv/example/public
index = index.html
serve_static_root = true
proxy = http://127.0.0.1:3000
